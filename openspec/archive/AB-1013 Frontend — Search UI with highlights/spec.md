## Summary

Adds keyword search to the `/notes` page. A debounced search input sits above the notes grid; when a query is active the URL becomes `/notes?q=...` and the grid switches from normal note cards to ranked `SearchResultCard` components that render the backend's `highlight` HTML snippet (with `<mark>` tags) inline. Tag sidebar filters continue to work while search is active. Clearing the input removes `?q=` and reverts to the normal notes list. All state is URL-driven so the browser back button and page reloads work correctly.

---

## In Scope

- Search input on the `/notes` page, above the notes grid
- 400ms debounced keystroke search (no explicit submit required)
- URL state via `?q=`, `?page=`, `?tagId[]=` ‚Äî all three params coexist
- `useSearch` hook wrapping `GET /api/search`
- `SearchResultCard` component rendering note title, tags, date, and `highlight` HTML via `dangerouslySetInnerHTML`
- Tag sidebar filters forwarded to the search request when `?q=` is active
- Pagination controls reused for search results; page resets to 1 on query change
- Loading skeletons during pending search
- Empty results state with query name and clear affordance
- Clearing the input (debounce fires on empty value) removes `?q=` and restores the notes list + sort controls
- Sort controls hidden while search mode is active (results are ranked by relevance)
- URL persistence: `/notes?q=foo` pre-fills and fires search on load
- `useDeleteNote` invalidates `['search']` query cache entries in addition to `['notes']`

## Out of Scope

- Search bar in the Navbar
- Dedicated `/search` route
- Search autocomplete or suggestions
- Search history
- Sort controls while in search mode
- Searching from the note editor page (`/notes/:id`)
- Backend changes of any kind

### URL state model

```
No query active:
  /notes?page=1&sortBy=updatedAt&sortDir=desc&tagId[]=abc
  ‚Üí useNotes({ page, sortBy, sortDir, tagId })
  ‚Üí NoteCard grid + sort <Select>

Query active:
  /notes?q=react&page=1&tagId[]=abc
  ‚Üí useSearch({ q: 'react', page: 1, limit: 20, tagId: ['abc'] })
  ‚Üí SearchResultCard grid; sort <Select> hidden
```

### Debounce implementation

`NotesPage` holds a local `rawQuery` via `useState` bound to the search input. A `useEffect` runs a `setTimeout(400ms)` on `rawQuery`; the cleanup function clears the timer on each new keystroke. When the timeout fires, the trimmed value is written to `searchParams` (setting or deleting `?q=` and resetting `?page=1`). The debounced URL param ‚Äî not the raw input ‚Äî is what `useSearch` reads, so TanStack Query only sees stable keys.

### Highlight XSS surface

The `highlight` field is generated exclusively by PostgreSQL's `ts_headline` function using a fixed template (`StartSel=<mark>, StopSel=</mark>`). The injected HTML is always `<mark>‚Ä¶</mark>` wrapping fragments of the authenticated user's own note content. Server-side auth ensures users can only receive highlights from notes they own. No third-party content reaches this field. `dangerouslySetInnerHTML` is acceptable under these constraints.
