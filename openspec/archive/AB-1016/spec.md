## Summary

Implements the Playwright E2E test suite mandated by FRS §5.4.3 ("Essential flows must be fully validated using automated E2E tests"). The suite lives in a root-level `e2e/` directory and exercises the full user journey through the browser: registration, login, auth guards, logout, note creation and autosave, full-text search, tag management, secure share link generation and anonymous access, and version history with restore. A global setup step resets the test database, seeds a fixed user, and persists Playwright `storageState` so authenticated tests bypass the login UI. All tests run against Chromium headless. The `BASE_URL` env var controls the target host, enabling both local and CI execution without config changes.

---

## In Scope

- Root-level `e2e/` workspace with its own `package.json` and `playwright.config.ts`
- Global setup: test DB reset via `prisma migrate reset`, fixed user seed via API, storageState persisted to `e2e/.auth/user.json`
- **Auth journey:** register (new user via UI), login (valid + invalid credentials), auth guard redirect, guest route redirect, logout
- **Notes journey:** create note, edit title and content, autosave confirmation ("Synced" indicator)
- **Search journey:** full-text match with highlighted snippet, no-results empty state
- **Tags journey:** create tag, attach tag to note, filter notes list by tag
- **Share journey:** generate share link, anonymous view in a new browser context (no storageState), revoke link and verify access denied
- **Version history journey:** version entry visible after autosave, restore to an earlier snapshot
- Chromium headless only
- `BASE_URL` env var for flexible host targeting

## Out of Scope

- Firefox and WebKit browser targets
- Forgot password / OTP reset UI flow
- Soft delete and trash recovery flow
- Pagination and sorting controls
- Mobile viewport testing
- Performance or visual regression testing
- Backend unit and integration tests (fully covered by AB-1001–AB-1009)

## API Contract

No new API endpoints are introduced. The E2E suite exercises all existing endpoints through browser UI interactions. `global.setup.ts` calls the following endpoints directly via Playwright's `APIRequestContext`:

- `POST /api/auth/register` — create fixed E2E seed user
- `POST /api/auth/login` — obtain access token; state saved to `e2e/.auth/user.json`

---

## Database Changes

None. The E2E suite targets the existing schema on `TEST_DATABASE_URL`. The global setup runs:

```bash
pnpm --filter backend prisma migrate reset --force --skip-seed
```

This is a test-environment-only operation, never run against `DATABASE_URL`.

## Architecture Notes

### Workspace layout

```
e2e/
├── package.json              { "@playwright/test", "prisma" devDependencies }
├── playwright.config.ts      projects: [setup, chromium]; storageState; baseURL
├── global.setup.ts           DB reset → register seed user → login → save .auth/user.json
├── .auth/                    gitignored; contains user.json storageState
└── tests/
    ├── auth.spec.ts          S1–S6 (register, login, invalid login, guards, logout)
    ├── notes.spec.ts         S7–S8 (create note, autosave)
    ├── search.spec.ts        S9–S10 (full-text match, no results)
    ├── tags.spec.ts          S11–S13 (create tag, attach, filter)
    ├── share.spec.ts         S14–S16 (generate link, anonymous view, revoke)
    └── versions.spec.ts      S17–S18 (version list, restore)
```

### Playwright project configuration

`playwright.config.ts` defines two projects:

```
setup     → runs global.setup.ts; no storageState dependency
chromium  → depends on setup; all spec files use storageState: 'e2e/.auth/user.json'
```

Tests in `auth.spec.ts` that verify unauthenticated behavior explicitly create a new browser context with no storageState (or call `page.context().clearCookies()` + clear localStorage) per individual test.

### Anonymous browser context pattern

In `share.spec.ts`:

```typescript
const anonCtx = await browser.newContext(); // no storageState
const anonPage = await anonCtx.newPage();
await anonPage.goto(shareUrl);
// assert content / error
await anonCtx.close();
```

### Data setup within tests

`notes.spec.ts`, `tags.spec.ts`, `share.spec.ts`, and `versions.spec.ts` create required pre-existing data in `beforeAll` using `request.post(...)` (Playwright's `APIRequestContext` inheriting the seeded user's auth from storageState). Tests within a `describe` block build on this shared state; no state crosses spec file boundaries.

### Root workspace integration

`e2e/` is added to `pnpm-workspace.yaml`. Two new scripts are added to the root `package.json`:

```json
"e2e":    "pnpm --filter e2e playwright test",
"e2e:ui": "pnpm --filter e2e playwright test --ui"
```

`BASE_URL` is read in `playwright.config.ts` as `process.env.BASE_URL ?? 'http://localhost:5173'`.
